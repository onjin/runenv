fix(cli): replace "Environment file None does not exist" with context-aware message

When --env-file is omitted and auto-detection fails, the error now reads
"No .env / .env.json / .env.toml / .env.yaml found in <cwd>" instead of
"Environment file None does not exist". When an explicit --env-file is
given but missing, the filename is shown in backticks. Covers run, list,
and lint subcommands.
