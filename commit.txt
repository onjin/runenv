docs(parser): document and test ${VAR} fallback to os.environ

The fallback is intentional: env files may reference parent-process
variables (e.g. HOME, USER) without redeclaring them. Resolution order
is now documented in the substitute_variables docstring:
  1. keys from the env file
  2. os.environ
  3. empty string (undefined refs expand silently)

Five new tests in TestVariableSubstitution cover all three cases plus
direct calls to substitute_variables.
